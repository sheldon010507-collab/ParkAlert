## 5. 核心功能详细设计

### 5.1 抄牌员标记

用户在当前位置提交标记，填写以下字段：

|字段   |类型      |必填|说明                                       |
|-----|--------|--|-----------------------------------------|
|抄牌员类型|单选      |✅ |Council Warden / Private Parking / Police|
|人数   |单选      |✅ |1人 / 2人 / 多人                             |
|移动方向 |8方向箭头 UI|✅ |N / NE / E / SE / S / SW / W / NW        |
|移动方式 |单选      |✅ |步行 / 骑车 / 开车                             |
|位置   |GPS 自动获取|✅ |允许微调                                     |
|时间戳  |系统自动    |✅ |提交时间，用户可见                                |

**自动过期规则（不可由用户修改）：**

|移动方式|过期时间 |
|----|-----|
|步行  |60 分钟|
|骑车  |45 分钟|
|开车  |30 分钟|

### 5.2 停车位标记

- 每用户同时只能激活 **1 个**停车位标记
- 新标记自动覆盖旧标记
- 警戒半径：**固定 150m**（MVP 阶段不可自定义）
- 用户手动关闭，或 GPS 判断已离开后自动关闭

### 5.3 警报系统

**触发条件：** 抄牌员标记进入用户停车位 150m 警戒范围

**MVP 触发方式（仅前台）：**

- App 在前台：弹窗 + 震动
- 后台通知：二期实现

**警报内容示例：**

```
⚠️ Council Warden × 2 spotted 120m from your car, heading your way
Posted 3 mins ago
```

**防骚扰机制：** 同一标记 15 分钟内不重复触发

### 5.4 账号系统

- 注册 / 登录：Email + 密码（Supabase Auth）
- 社交登录（Apple ID / Google）：二期
- 未登录用户无法查看任何内容，强制跳转注册页

## 6. 数据库设计

```sql
-- 用户扩展表（基于 Supabase Auth）
profiles
  id            uuid references auth.users  PRIMARY KEY
  username      text
  reputation    int       DEFAULT 100
  created_at    timestamptz DEFAULT now()

-- 抄牌员标记
warden_sightings
  id            uuid      PRIMARY KEY DEFAULT gen_random_uuid()
  user_id       uuid      references profiles
  lat           float     NOT NULL
  lng           float     NOT NULL
  warden_type   text      NOT NULL   -- 'council' | 'private' | 'police'
  count         text      NOT NULL   -- 'one' | 'two' | 'multiple'
  direction     text      NOT NULL   -- 'N'|'NE'|'E'|'SE'|'S'|'SW'|'W'|'NW'
  movement      text      NOT NULL   -- 'walking' | 'cycling' | 'driving'
  expires_at    timestamptz NOT NULL -- 系统根据 movement 自动计算
  created_at    timestamptz DEFAULT now()

-- 用户停车位（每用户只保留一条）
parked_cars
  id            uuid      PRIMARY KEY DEFAULT gen_random_uuid()
  user_id       uuid      references profiles UNIQUE  -- UNIQUE 保证只有一条
  lat           float     NOT NULL
  lng           float     NOT NULL
  radius_m      int       DEFAULT 150
  created_at    timestamptz DEFAULT now()
```

## 7. 技术架构

|层               |技术选型                           |说明                           |
|----------------|-------------------------------|-----------------------------|
|移动端框架           |React Native + Expo            |iOS 优先，你熟悉的技术栈               |
|平台              |iOS only（MVP）                  |TestFlight 内测，无需 App Store 审核|

|认证              |Supabase Auth（Email）           |MVP 最简方案                     |
|实时数据            |Supabase Realtime              |无需额外服务                       |
|推送通知            |Expo Notifications             |MVP 阶段仅前台                    |
|定位              |expo-location（前台）              |后台定位为二期                      |
|Landing Page（二期）|Next.js + Vercel               |MVP 完成后再做                    |
|邮件（二期）          |Resend 或 Loops.so              |等待名单邮件                       |
|广告（二期）          |Google AdMob                   |变现阶段接入                       |

## 9. 冷启动策略

### 9.1 核心问题

新用户打开空地图会立即流失。**MVP 发布前必须完成数据播种。**

### 9.2 Glasgow 数据播种（发布前 2 周）

|行动  |说明                            |
|----|------------------------------|
|人工巡查|每天在 Glasgow 市中心记录抄牌员出没位置和时间规律 |
|热点图层|根据巡查数据生成”高风险区域”图层，无实时标记时仍有参考价值|
|目标  |发布第一天地图上至少有 20 个历史热点标记可见      |

### 9.3 种子用户获取渠道（第 1 个月）

|渠道                      |具体行动           |预期人数         |
|------------------------|---------------|-------------|
|Edinburgh / Glasgow 本地朋友|直接邀请 TestFlight|20–30 人      |
|Sichuan House 餐厅顾客      |收银台卡片 + 等位区海报  |50–80 人      |
|UK 华人微信群                |中英双语介绍，邀请码限量   |30–50 人      |
|r/Glasgow / r/Edinburgh |真实故事帖，非广告      |40–60 人      |
|Glasgow Facebook 司机群    |加入社群，自然分享      |20–40 人      |
|**合计**                  |               |**160–260 人**|

### 9.4 种子用户激励

- **Founding Spotter 徽章：** 前 200 名用户永久显示创始成员标识
- **直接反馈通道：** 建立微信 / WhatsApp 群，开发者亲自回复每条反馈
- **周报邮件：** 每周告知用户”本周共有 X 次预警，帮助用户避开 X 次罚款”

### 10.1 目标

- 收集等待名单（目标 200 人）
- 验证市场需求数据
- 建立品牌第一印象

### 10.2 页面结构

**Hero Section**

- 标题：*“Glasgow’s parking wardens are out there. Now you’ll know before they reach you.”*
- CTA：**“Join the Waitlist — It’s Free”**

**痛点 Section（3 张卡片）**

- 💸 *“Average PCN fine: £70. That’s a lot of lunches.”*
- 😤 *“Wardens don’t announce themselves. But your neighbours can.”*
- 📍 *“Know exactly where they are, before they reach your car.”*

- 字段：邮箱 + 城市（Glasgow / Edinburgh / Aberdeen / Other UK）

## 12. 法律与合规（UK）

|项目          |要求                       |
|------------|-------------------------|
|GDPR        |位置数据属敏感数据，需明确隐私政策和数据处理协议 |
|抄牌员信息       |不得收集照片、姓名等个人信息，只记录位置和行为特征|
|Terms of Use|明确平台为社区信息共享工具，不构成法律建议    |
|数据存储        |位置数据存储于 UK / EU 服务器      |
|App Store 措辞|强调”社区信息共享”，避免被认定为”协助逃避执法”|
